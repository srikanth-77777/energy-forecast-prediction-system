1. Title & Problem Identification
1.1 Project Title
A Systematic Study on Data-Centric Refinement for Urban Energy Demand Forecasting using Time-Series Feature Engineering and Machine Learning
1.2 Problem Identification
Urban areas face continuously fluctuating electricity demand driven by population growth, climate variability, industrialization, and daily human behavioral patterns. This makes accurate energy demand forecasting a critical challenge for power distribution companies and smart city planners.
Traditional forecasting approaches focus primarily on improving model architecture complexity (deeper neural networks, more parameters) while neglecting the quality and richness of input features derived from raw time-series data. This leads to:
Inaccurate predictions due to noisy or incomplete time-series data
Missing temporal patterns such as seasonal trends, peak hours, and holiday effects
Poor generalization of models trained on unrefined, raw energy readings
Inefficient energy distribution leading to waste or shortages
This project addresses the problem by demonstrating that data-centric refinement — systematic cleaning, preprocessing, and feature engineering of time-series energy data — significantly outperforms model-centric improvements when it comes to forecasting accuracy.
2. Research Gap
While numerous studies apply deep learning and ensemble models to energy demand forecasting, a critical research gap exists:
Most existing work treats feature engineering as a secondary concern, focusing instead on architectures like Transformers, BiLSTM, or hybrid CNN-LSTM models
There is limited systematic analysis of how individual feature types (lag features vs. rolling statistics vs. time-based features) contribute to prediction accuracy
Few studies benchmark data-centric improvements against model-centric improvements on the same urban energy dataset
The impact of data preprocessing choices (outlier removal, normalization method, resampling strategy) on downstream model performance is under-studied
Most public datasets used are climate-specific (weather-driven) and do not isolate the effect of temporal pattern extraction on urban consumption
This project fills this gap by conducting a structured ablation study where the same set of models (Linear Regression, Random Forest, Gradient Boosting, LSTM) are evaluated with progressively enriched feature sets to quantify the contribution of each data-centric refinement step.

6.2 Core Algorithm: XGBoost Objective
XGBoost minimizes a regularized objective combining a loss function and a complexity penalty:
Objective(θ) = Σ l(yᵢ, ŷᵢ) + Σ Ω(fₖ)  where  Ω(f) = γT + ½λ||w||²
Here: l = loss function (MSE for regression), T = number of leaves, w = leaf weights, γ and λ = regularization parameters to prevent overfitting.

Step 4: Feature Engineering — Time-Based Features
1
Extract hour of day: df['hour'] = df.index.hour
2
Extract day of week: df['day_of_week'] = df.index.dayofweek
3
Extract month: df['month'] = df.index.month
4
Extract year: df['year'] = df.index.year
5
Create is_weekend flag: df['is_weekend'] = df['day_of_week'].isin([5, 6]).astype(int)
6
Create is_peak_hour flag (6 AM–10 PM): df['is_peak'] = df['hour'].between(6, 22).astype(int)
7
Create season feature (0=winter, 1=spring, 2=summer, 3=fall) based on month
8
Add quarter feature: df['quarter'] = df.index.quarter
Step 5: Feature Engineering — Lag Features
1
Create 1-hour lag: df['lag_1h'] = df['PJME_MW'].shift(1)
2
Create 24-hour lag: df['lag_24h'] = df['PJME_MW'].shift(24)
3
Create 168-hour (1 week) lag: df['lag_168h'] = df['PJME_MW'].shift(168)
4
Create 2-hour lag and 6-hour lag for short-term patterns
5
Create lag difference features: df['diff_1h'] = df['PJME_MW'].diff(1)
6
Create percent change feature: df['pct_change_24h'] = df['PJME_MW'].pct_change(24)
7
Drop rows with NaN values introduced by lag features
8
Verify lag features using correlation heatmap
Step 6: Feature Engineering — Rolling Statistics
1
Rolling mean (24h window): df['roll_mean_24h'] = df['PJME_MW'].rolling(24).mean()
2
Rolling standard deviation (24h): df['roll_std_24h'] = df['PJME_MW'].rolling(24).std()
3
Rolling minimum (168h): df['roll_min_168h'] = df['PJME_MW'].rolling(168).min()
4
Rolling maximum (168h): df['roll_max_168h'] = df['PJME_MW'].rolling(168).max()

9. Conclusion
This project demonstrates that a data-centric approach to urban energy demand forecasting — focusing on systematic feature engineering, preprocessing, and temporal pattern extraction — yields significantly more accurate forecasting results than simply increasing model complexity.
Key contributions of this work include:
A structured 50-step implementation pipeline covering data acquisition through model deployment
Systematic comparison of 4 forecasting algorithms on the same enriched feature set
Quantification of the contribution of each feature engineering technique (lag, rolling, time-based) to prediction accuracy
A reproducible, well-documented implementation using publicly available datasets
The findings of this study have practical implications for smart city energy management, load balancing, and sustainable power distribution in urban environments.
